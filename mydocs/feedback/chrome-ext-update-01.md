# 기능 추가 요청: What's New 팝업 (v0.X.X 배포용)

## 목적
사용자가 rhwp 업데이트를 인지하고 "개발자가 반응하고 있다"를 
느끼게 하는 것. 자동 업데이트는 조용해서 사용자가 변화를 모를 수 있음.

## 사용자 스토리
- As a user, 업데이트된 rhwp를 처음 열었을 때
- I want to know 이번 버전에서 무엇이 달라졌는지
- So that 내가 리포트하거나 경험한 문제가 반영되었는지 확인할 수 있음

## 설계 방향
- 침해적이지 않은 배지 방식 (시나리오 A)
- 사용자가 확인 후 같은 버전에서는 다시 안 뜸
- 30초 무반응 시 배지만 자동 사라짐

## UI/UX 사양
### 표시 위치
- rhwp 에디터 열린 직후 화면 우측 상단 (또는 기존 UI와 충돌 없는 자리)

### 초기 상태: 작은 배지
- 텍스트: "✨ v{version} 업데이트"
- 클릭 시 팝업 확장
- 30초 무반응 → 배지 페이드아웃

### 확장된 팝업 내용
- 헤더: "rhwp v{version} 업데이트"
- 본문: 
  - 이번 버전 변경 사항 목록 (3~5개)
  - 알려진 한계 (현재 버전은 HWPX 저장 미지원)
  - 외부 기여자 감사 (이번 버전에 기여가 있을 경우)
- 액션:
  - [자세히 보기] → GitHub Release 페이지 새 탭 열기
  - [닫기] → 팝업 닫기 + 현재 버전 저장

## 기술 요구사항
- chrome.storage.local로 lastSeenVersion 관리
- 서버 통신 없음 (로컬 처리만)
- manifest.json의 version과 비교
- 변경 사항 내용은 하드코딩 or 별도 JSON 파일 (선택)
