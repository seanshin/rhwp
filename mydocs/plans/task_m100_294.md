## 1. 목표

KTX.hwp 목차의 교차-run 오른쪽 탭(tab_type=1) 처리에서 **자릿수에 무관하게 right edge가 tab_pos_abs에 정렬**되도록 수정한다.

## 2. 현상 및 근본 원인

### 2.1 현재 SVG 관측값 (ee65fa7 = 3c06781 leading_ws 상태)

KTX.hwp 목차 페이지 (`-p 1` 기준, tab_pos_abs = col_area.x + tab_pos = **699.76px**):

| 페이지번호 | 첫 자리 x | right edge | 목표 | 오차 |
|-----------|----------|------------|------|------|
| 단일 '3' (장제목) | 699.76 | 709.09 | 699.76 | **+9.33px** |
| 단일 '8' (장제목) | 699.76 | 709.09 | 699.76 | **+9.33px** |
| 단일 '4','6','7','9' (소제목) | 699.88 | 709.21 | 699.76 | **+9.45px** |
| 두 자리 '14','17' 등 (소제목) | 699.88 | 718.54 | 699.76 | **+18.78px** |
| 두 자리 '16','20','24' (장제목) | 700.09 | 718.76 | 699.76 | **+19.00px** |

단일 자리는 약 9.33px(1 digit width), 두 자리는 약 18.66px(2 digit width) 만큼 오른쪽으로 밀려 있다.

### 2.2 근본 원인

`src/renderer/layout/paragraph_layout.rs`의 `pending_right_tab_render` 처리 (3c06781):

```rust
1 => {
    let trimmed = run.text.trim_start();
    let leading_ws = &run.text[..run.text.len() - trimmed.len()];
    let leading_ws_w = if leading_ws.is_empty() { 0.0 }
                       else { estimate_text_width(leading_ws, &text_style) };
    x = col_area.x + tab_pos - leading_ws_w;
}
```

이 공식은 **leading whitespace만 역방향 이동**한다.

- run B = "3" (선행 공백 없음) → `x = tab_pos_abs - 0 = 699.76`  
  → "3" 자체가 699.76에서 시작 → right edge = 709.09 ✗
- run B = " 3" (선행 공백 있음) → `x = tab_pos_abs - space_w ≈ 695.09`  
  → 공백 후 "3"이 699.76에서 시작 → right edge = 709.09 ✗

**올바른 공식**: x = tab_pos_abs − (run B 이후 모든 run의 총 폭)

이렇게 하면:
- "3" (폭 9.33) → `x = 699.76 - 9.33 = 690.43` → right edge = 699.76 ✓
- "14" (총 폭 18.66) → `x = 699.76 - 18.66 = 681.10` → right edge = 699.76 ✓

## 3. 수정 방향

### 3.1 대상 파일 및 위치

`src/renderer/layout/paragraph_layout.rs` — `pending_right_tab_render.take()` 블록 내 `tab_type == 1` 분기

### 3.2 수정 전 (현재 3c06781)

```rust
if let Some((tab_pos, tab_type)) = pending_right_tab_render.take() {
    text_style.line_x_offset = x - col_area.x;
    let next_w = estimate_text_width(&run.text, &text_style);
    match tab_type {
        1 => {
            let trimmed = run.text.trim_start();
            let leading_ws = &run.text[..run.text.len() - trimmed.len()];
            let leading_ws_w = if leading_ws.is_empty() { 0.0 }
                               else { estimate_text_width(leading_ws, &text_style) };
            x = col_area.x + tab_pos - leading_ws_w;
        }
        2 => x = col_area.x + tab_pos - next_w / 2.0,
        _ => {}
    }
}
```

### 3.3 수정 후 (look-ahead)

```rust
if let Some((tab_pos, tab_type)) = pending_right_tab_render.take() {
    text_style.line_x_offset = x - col_area.x;
    let next_w = estimate_text_width(&run.text, &text_style);
    match tab_type {
        1 => {
            // 현재 run + 이후 모든 run의 총 폭 합산 → right edge = tab_pos_abs
            let mut total_w = next_w;
            for fr in &comp_line.runs[(run_idx + 1)..] {
                let mut fts = resolved_to_text_style(styles, fr.char_style_id, fr.lang_index);
                fts.default_tab_width = tab_width;
                fts.tab_stops = tab_stops.clone();
                fts.auto_tab_right = auto_tab_right;
                fts.available_width = available_width;
                total_w += estimate_text_width(&fr.text, &fts);
            }
            x = col_area.x + tab_pos - total_w;
        }
        2 => x = col_area.x + tab_pos - next_w / 2.0,
        _ => {}
    }
}
```

## 6. 위험 및 트레이드오프

### 6.1 look-ahead 범위

`comp_line.runs[(run_idx + 1)..]`는 동일 줄의 모든 후속 run을 포함한다. 목차 항목 구조는 `[제목\t][숫자]` 또는 `[제목\t][ ][숫자]`이므로 후속 run이 페이지번호 외 다른 내용을 포함하지 않는다. 일반 문단에서 오른쪽 탭 이후 다른 run이 존재할 경우에도 right edge 정렬이 유지되므로 범위 제한 불필요.

### 6.2 기존 `cargo test` 영향

3c06781에서 golden SVG가 업데이트된 경우, right edge 좌표가 변경되므로 해당 golden snapshot을 재생성해야 할 수 있다.

```bash
UPDATE_GOLDEN=1 cargo test --test svg_snapshot
```

### 6.3 이전 look-ahead 시도 (1a8e91a)와의 차이

1a8e91a에서 같은 look-ahead 방식이 적용되었으나 사용자가 "아직 조정안됨" 확인 후 롤백. SVG 분석상 right edge는 올바른 값(~699.76)으로 계산되었으므로, 당시 문제는 **배포 미완료 또는 브라우저 캐시**로 추정된다.

이번 구현:
- 코드 변경: `tab_type == 1` 분기만 교체, 다른 누적 변경 없음
- 배포: `wasm-pack` → `build-editor.sh` → `scp` → `pm2 restart` 순서 명시적 확인
