## 2. 문제 현황

### 2.1 리더 도트 렌더링 (확인된 버그)

`src/renderer/svg.rs` L1766:
```rust
match leader.fill_type {
    3 => {  // 점선 ···
        "<line ... stroke-width=\"0.5\" stroke-dasharray=\"1 2\"/>"
        // stroke-linecap 없음 → 사각 끝, 시각적으로 얇은 대시선
    }
```

- `fill_type=3` = 점선(···), 한컴은 원형 점 렌더
- `stroke-linecap="round"` + `stroke-dasharray="0 gap"` + `stroke-width` 조정으로 수정 가능
- KTX.hwp 목차는 fill_type=3 사용 확인됨 (SVG 출력에서 `stroke-dasharray="1 2"` 관찰)

### 2.2 소제목 페이지 번호 탭 위치 (조사 필요)

현재 SVG에서 관찰:
- 장제목(ps_id=109): 리더 x2=694.76, 페이지번호 right edge ≈ x=719px
- 소제목(ps_id=111): 리더 x2=670.68, 페이지번호 right edge ≈ x=699px (약 20px 차이)

**원인 가설**:
- **A) 의도된 차이**: ps_id=111이 left_margin indent를 가지며 right tab이 단락 text area 기준으로 계산 → 절대좌표 차이는 정상 (한컴도 동일)
- **B) 버그**: right tab 위치 계산에 indent가 잘못 적용되어 페이지번호가 밀림 → 수정 필요

→ Stage 1에서 ps_id=109/111의 탭 정의 + margin을 덤프하고 한컴 비교로 확정

## 3. 수정 방향

### 방안 A (fill_type=3 점선 개선)

```rust
3 => {  // 점선 ··· — round dots
    "<line x1=\"{}\" y1=\"{}\" x2=\"{}\" y2=\"{}\"
     stroke=\"{}\" stroke-width=\"1.5\"
     stroke-dasharray=\"0 {gap}\"
     stroke-linecap=\"round\"/>"
    // gap ≈ 3.0 (글꼴 크기에 비례, 한컴 점 간격 맞춤)
}
```

- `stroke-dasharray="0 gap"`: 점 크기 0 + 간격으로 원형 점 표현
- `stroke-linecap="round"`: 선 끝을 원형으로 → 실제 점(·)처럼 보임
- `stroke-width=1.5`: 점 직경

### 방안 B (소제목 탭 위치, 조사 결과에 따라)

- 원인 A (의도된 차이) 확인 시: 수정 불필요
- 원인 B (버그) 확인 시: right tab 계산 시 paragraph indent 처리 로직 수정

## 6. 위험 / 트레이드오프

- **fill_type=3 점 간격**: 글자 크기에 따라 최적 gap이 달라짐. 고정 gap(예: 3.0px)으로 충분한지 확인 필요. 글자크기 비례 gap(font_size × 0.15 등) 검토.

- **다른 fill_type**: fill_type=1(실선), 2(파선) 등은 이번 범위에서 변경하지 않음.
