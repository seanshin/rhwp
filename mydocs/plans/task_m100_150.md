## 3. 수정 방향

### 3.1 SVG 필터 수식

brightness (`b`, i8 ∈ [-127, 127]) 와 contrast (`c`, i8 ∈ [-127, 127]) 를 단일 `feComponentTransfer` 로 결합:

```
slope     = (127 + c) / 127.0          // c=0 → 1.0 (원본), c>0 → 강조, c<0 → 약화
intercept = slope * (b / 255.0) + 0.5 * (1.0 - slope)
```

적용 결과 (`feComponentTransfer type="linear"`):
```
output = slope × input + intercept
```

- b=0, c=0 → slope=1.0, intercept=0.0 → 원본과 동일
- b=50, c=0 → 밝기만 +50/255 ≈ +20% 오프셋
- b=0, c=50 → 대비 강조 (slope=1.39)
- b=30, c=20 → 밝기+대비 동시 적용

### 3.2 필터 ID 체계

값 조합마다 고유 ID: `rhwp-img-bc-{b}_{c}`  
(예: `rhwp-img-bc-30_20`, `rhwp-img-bc-0_-50`)

효과 필터(그레이스케일 등)와 병용 시 체인:
```xml
<filter id="rhwp-img-gs-bc-30_20">
  <!-- grayscale feColorMatrix → result="gs" -->
  <!-- feComponentTransfer in="gs" → brightness/contrast -->
</filter>
```

## 6. 범위 제외 — 워터마크

워터마크(`ImageEffect::Watermark` 또는 별도 필드)는 HWP 파서·모델에 미구현 상태. 분석 및 파서 추가가 필요하므로 이번 타스크 범위 밖. 별도 이슈로 분리 권고.
