타스크 진행 시 프로세스를 절대 생략하지 않는다.

**Why:** #61에서 이슈 등록 후 바로 구현에 들어가면서 타스크 브랜치, 오늘할일 등록, 구현 계획서를 모두 누락했다. Hyper-Waterfall의 핵심 원칙(거시적 워터폴 규율)을 스스로 어긴 사례.

**How to apply:** 모든 타스크는 반드시 다음 순서를 따른다:
1. GitHub Issue 등록
2. 타스크 브랜치 생성 (`local/task{N}`)
3. 오늘할일 문서에 등록
4. 구현 계획서 작성 → 작업지시자 승인
5. 구현 → 테스트 → 커밋
6. 완료 보고 → 이슈 클로즈

빠르게 고칠 수 있는 버그라도 프로세스를 건너뛰지 않는다.
